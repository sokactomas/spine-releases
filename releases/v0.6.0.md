# Spine v0.6.0

Released: 2026-07-05

## What's new

Spine's biggest release yet: the app becomes a **personal knowledge OS for your
AI coding CLIs** - a passive menu bar layer that decides what to remember, what
to forget, and what context every new session starts with.

- **Spine moved to the menu bar.** A glyph in the bar with a small glass panel:
  health, your compounding advantage, the per-project resume point, one-click
  Undo. The built-in terminal is gone - use Ghostty, iTerm, anything; capture
  and the MCP hub never depended on it. Launching Spine no longer opens a
  window; it just starts running (and can start at login).
- **Changes merge themselves.** The Review queue is gone. Adds, edits and
  deletions from Claude/Codex flow into the shared base automatically - newest
  wins, git history keeps everything, the freshest merge is one click to undo.
  The base then re-projects to every CLI on its own.
- **Memory that reads itself back.** Whenever the graph learns (a distilled
  session, an auto-merge, a `spine_remember` from any CLI), the
  CLAUDE.md/AGENTS.md context block refreshes with the notes most relevant -
  by meaning, not keywords - to the project and files being changed right now.
- **…and forgets on purpose.** Old isolated notes and skills nobody used
  quietly age out (Undo restores them; corrections and decisions are never
  forgotten). `spine_search` also stopped surfacing superseded notes.
- **The right model for the task.** A routing policy rides in every session's
  context: orchestrate on the session model, delegate heavy work to a stronger
  worker via subagents, escalate hard judgment calls to the advisor
  (Settings → Advisor → Task routing).
- **Skill context-load accounting.** The Skill library shows what your skills
  cost per request and flags heavyweight descriptions.
- **A new face.** New app + menubar icon (the layers mark), a System-Settings
  style main window, real glass with a Liquid glass slider, and completion
  chimes that now work from any terminal (update the statusline hook in
  Settings → Skill library → Install and CLI hooks).

Under the hood: the Go sidecar dropped the whole PTY subsystem (−3,600 lines,
no shell is ever spawned), and the context pipeline is CLI-agnostic - memory
files as the push channel, MCP as the pull channel, identical for Claude and
Codex.

## Install

1. Download the `Spine_0.6.0_universal.dmg` asset from GitHub Releases.
2. Open the DMG and drag Spine to Applications.
3. Launch Spine - it appears in the menu bar (no window; that's the point).

# Spine v0.6.1

Released: 2026-07-07

## What changed

This release tightens the new menu-bar Spine flow so it behaves like the passive
shared layer it is meant to be: less setup, fewer manual queues, and cleaner
Codex/Claude parity.

- **Advisor is automatic now.** `spine_advisor` routes by the CLI that asked:
  Codex sessions use Codex, Claude sessions use Claude. You can still pin a
  provider/model in Settings, but the default no longer asks you to choose.
- **Skills are passive, not a chore list.** The old gap inbox is now framed as an
  auto-learn queue. Spine records uncovered asks from session distills, builds
  when it can act safely, and leaves Settings as an audit/manual override
  surface rather than a queue you need to clear.
- **Codex skill projection is safer.** Spine keeps Codex on a capped native
  `~/.codex/skills` subset and fixes agent+skill name collisions inside
  `SKILL.md`, so Codex sees unique skill names instead of duplicate frontmatter.
- **The visual identity is consistent.** App icons, the menu-bar template icon,
  favicon and About mark all use the same high-contrast spine glyph.
- **Less leftover setup/debug noise.** Old popover debug logging and advisor
  setup copy were removed; Settings now describes the passive behavior Spine
  actually uses.

## Install

1. Download the `Spine_0.6.1_universal.dmg` asset from GitHub Releases.
2. Open the DMG and drag Spine to Applications.
3. Launch Spine - it appears in the menu bar and auto-wires the MCP hub into
   installed CLIs.
4. Restart running Claude/Codex sessions so they reload the updated context and
   skill projections.
